> 對應 [Part 26 進階機器學習](../chapters/26-advanced-ml/README.md)。核心:集成學習、聚類、PCA、超參數調校、不平衡處理、可解釋性。

## Q1. 決策樹怎麼學?為什麼極易過擬合?

**考點**:決策樹([01-decision-trees](../chapters/26-advanced-ml/01-decision-trees.md))

**答**:決策樹用**遞迴二元分割**——每次選「**降不純度(Gini/熵)最多**」的特徵與閾值,把資料切成越來越純的組,直到停止。

**極易過擬合**:不加限制的樹會**一直分割到每葉只剩一筆**——訓練準確率 100%(完美**記憶**),但學到的是雜訊不是規律,測試崩潰。**一定要剪枝**(`max_depth`、`min_samples_leaf`)。

**追問**:優點——**可解釋**(白箱規則,`export_text`)、**不需縮放**、能學非線性與交互;缺點——單棵易過擬合、不穩定(資料小變動樹就變),所以用**集成**改善(Q2)。

## Q2.(必考)集成學習的核心思想?bagging 和 boosting 差在哪?

**考點**:集成([02-ensemble-learning](../chapters/26-advanced-ml/02-ensemble-learning.md))

**答**:核心:**組合多個弱模型,錯誤相互抵消,整體更準更穩**(群體智慧)。兩大策略:

| | Bagging(隨機森林) | Boosting(梯度提升) |
|---|-------------------|---------------------|
| 方式 | **平行** | **序列** |
| 降低 | **變異** | **偏差** |
| 機制 | 平均多棵去相關的樹 | 每棵樹修正前面的錯 |

- **隨機森林降變異**:平均多個**去相關**的樹(各看隨機資料/特徵子集),獨立誤差相互抵消(N 個獨立誤差平均後變異 ~1/N)。
- **梯度提升降偏差**:每棵樹**擬合前面所有樹的殘差**(專攻答錯的),序列修正、逐步逼近。XGBoost/LightGBM 是其高效實現。

**追問**:**表格資料上集成樹常勝神經網路**;森林省心穩定好平行、提升更強但更易過擬合(要調 learning_rate)。

## Q3. k-means 怎麼運作?為什麼必須標準化?怎麼選 k?

**考點**:聚類([03-clustering](../chapters/26-advanced-ml/03-clustering.md))

**答**:k-means 迭代:**指派每點到最近中心 → 更新中心為群平均**,直到收斂,最小化組內平方和。

**必須標準化**:k-means 用**歐氏距離**——若不標準化,**大尺度特徵(收入)完全主宰距離**,小尺度(年齡)被忽略,聚類無意義。

**選 k**:

- **elbow(手肘法)**:畫 k vs inertia,找**急降後趨緩的拐點**(**不能選 inertia 最小**——它隨 k 單調降到每點一群)。
- **silhouette**:選分數最高的 k(綜合組內緊密 + 組間分離)。

**追問**:多次初始化(`n_init`)避免局部最優;k-means 假設**球形群**(非球形用 DBSCAN);聚類是探索性的(無標準答案)。

## Q4. PCA 在做什麼?為什麼降維前要標準化?

**考點**:PCA([04-pca](../chapters/26-advanced-ml/04-pca.md))

**答**:PCA 找**變異最大的正交方向(主成分)**,依變異排序,保留前 k 個降維——因為「變異大 = 資訊多」,丟棄變異小的(多是噪音/冗餘)。

**為何標準化**:PCA 對**變異**敏感,而變異依賴尺度——不標準化,大尺度特徵(收入,變異數十億)會**主宰主成分**(反映單位而非結構)。

**降維好處**:對抗維度災難、去冗餘/過擬合、視覺化(降到 2D)、加速。

**追問**:**解釋變異比例**(各 PC 承載的變異,累計決定保留多少維);用 `n_components=0.95` 自動選保留 95% 變異的維度;PCA 是**非監督**(不看標籤)、失去原特徵可解釋性。

## Q5. 參數和超參數差在哪?grid search 和 random search 怎麼選?

**考點**:超參數調校([05-hyperparameter-tuning](../chapters/26-advanced-ml/05-hyperparameter-tuning.md))

**答**:

- **參數**:模型**從資料學**的(係數、樹的分割點)。
- **超參數**:你**訓練前設定**的(`max_depth`、`C`、`learning_rate`)。

搜尋策略:**grid**(窮舉所有組合)、**random**(隨機取樣)、**Bayesian**(用前面結果智慧選)。

**random search 常勝 grid**:當**只有少數超參數真正重要**時,random search 用相同預算對重要參數的**搜尋密度更高**(grid 浪費在不重要的維度上)。**大空間用 random/Bayesian、小空間用 grid**。

**調參鐵律**:**全程用 CV 在 train 上、test 只最終評估一次**(用 test 調參 = 洩漏)。

**追問**:成本 = 組合數 × CV 折數(會爆炸);先廣後深(粗調再細調);選對 scoring。

## Q6.(必考)類別不平衡怎麼處理?

**考點**:不平衡([06-imbalanced-data](../chapters/26-advanced-ml/06-imbalanced-data.md))

**答**:不平衡時模型**偷懶偏多數類**(全猜多數類最省損失),recall 極低,準確率誤導(見 [Part 25 Q6](part25-machine-learning.md#q6必考準確率為什麼在不平衡資料上誤導precision-和-recall-差在哪))。三類手段:

1. **class_weight**:**放大少數類的損失權重**,逼模型重視(`class_weight="balanced"` 按頻率反比)——**最簡單常有效**。
2. **重採樣**:SMOTE 過採樣(合成少數類)/欠採樣(減多數類)——**只動訓練集**(測試集保持真實分布才能誠實評估)。
3. **閾值調整**:降低分類閾值提高 recall(不重訓)。

**追問**:重採樣在 CV 的**每折內**做(否則合成樣本洩漏);評估看 **P/R/F1/AUC 不看準確率**;依業務代價(漏抓 vs 誤報)調取捨。

## Q7. 特徵重要性怎麼看?內建和 permutation importance 差在哪?

**考點**:可解釋性([07-interpretability](../chapters/26-advanced-ml/07-interpretability.md))

**答**:可解釋性動機:法規、debug、發現偏見、獲取洞察(分全域/局部)。

- **內建特徵重要性(樹的 impurity-based)**:**快但有偏誤**——偏好高基數/連續特徵、且在**訓練資料**上算(可能反映過擬合)。
- **permutation importance**:**打亂某特徵看分數掉多少**——**模型無關、在 test 上算、更可靠**(反映對泛化的實際貢獻),不受基數偏誤影響。

**追問**:**SHAP**(基於博弈論,分解**單筆預測**的每特徵貢獻,全域 + 局部、理論嚴謹);相關特徵會互相掩蓋(permutation 的弱點);內在可解釋(線性/樹)vs 事後解釋(permutation/SHAP)。

## Q8. 進階 ML 專案的完整流程?

**考點**:進階 Capstone([08-capstone-advanced](../chapters/26-advanced-ml/08-capstone-advanced.md))

**答**:**split → 多模型 CV 選拔 → 優勝者調參 → 不平衡處理 → test 評估 → 可解釋**。

**先廣後深**:先用各架構的**合理預設**做 CV 比較(快速篩掉差的),再對**優勝架構深入調參**(兼顧全面與成本,別一開始就對每個架構完整調參——成本爆炸)。

**選架構與調參都用 CV、test 只最終一次**——守護泛化估計誠實(test 全程零接觸)。**不平衡貫穿全程**(指標用 f1、class_weight 從選拔就要對)。

**追問**:交付要附**可解釋**(permutation importance/SHAP,在 test 上算);追求可信可解釋而非炫技高分。
